Lecture 1 – How the Internet Works

Defence Course - Day 1

1.Basics

  • Internet = Network of computers sharing information.
  • Direct connection → Send 1s & 0s via cable (Ethernet/Fiber).
  • Problem of Scale: Too many devices → use Switch (Local Network).
  • Connecting Networks: Routers connect multiple local networks → Internet = "network of networks."

2.Addressing

  • Need unique ID for each device → IP Address.
  • IPv4: 32-bit (4.3B addresses). Shortage led to Public/Private IP.
  • Public IP: Unique on internet, given by ISP.
  • Private IP: Local, reused in home/office (192.168.x.x).
  • IPv6: 128-bit, huge supply (~340 undecillion addresses).

3.Data Transfer (Reliability)

  • Large files split into small packets.
  • Each packet has sender IP, receiver IP, sequence number.
  • Rules for sending/receiving = TCP/IP protocol.

4.MAC Address

  • Unique hardware ID (burned in every device).
  • Format examples: 3C:22:FB:A3:B4:C5 or 3c22.fba3.b4c5.
  • Used for local device identification.

5.Ports

  • Ports identify specific services on a device.
  • Range: 0–65535 (16-bit).
  • Categories:
    • Well-known (0–1023): HTTP(80), HTTPS(443), DNS(53), SSH(22).
    • Registered (1024–49151): MongoDB(27017), MySQL(3306), React dev(3000).
    • Dynamic (49152–65535): Temporary client ports.

6.DNS (Domain Name System)

  • Converts domain → IP address.
  • Steps:
    1. Check browser/OS/router cache.
    2. If not found, ask Recursive Resolver (ISP/Google 8.8.8.8).
    3. Root server → TLD server (.com, .org) → Authoritative server.
    4. IP returned & cached for faster use.
    5. Browser connects to that IP on correct port (e.g., 443).
