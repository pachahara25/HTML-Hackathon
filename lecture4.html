Lecture 4: File Path and Structure

Defence Course - Day 4

1. File Path System

  • Website = collection of files (HTML, CSS, JS, images, etc.).
  • Browser needs a file path to locate files.

1.1 Relative Paths

  • Same folder → about.html
  • Sub-folder → images/logo.png
  • Parent folder → ../images/logo.png

1.2 Absolute Paths

  • Full web address (http/https).
  • Used only for external resources.
  • Don’t use local paths like C:/Users/...

1.3 Why Relative is Safer

  • Browser runs in a secure Sandbox.
  • Prevents malicious websites from accessing local files.

1.4 Rules of Thumb

  • Own files → Use Relative paths.
  • External files → Use Absolute paths.

1.5 File System Differences

  • Windows: Root = C:\, separator = \
  • macOS/Linux: Root = /, separator = /

2. HTML Boilerplate

  • Standard template to start every HTML page.
  • Includes <!DOCTYPE html>, <html lang>, <meta charset>, <meta viewport>, <title>, <body>.
  • Ensures compatibility, mobile-friendliness, SEO.

2.1 Why Boilerplate is Important

  • Without → quirks mode, broken characters, bad mobile view.
  • With → predictable rendering, accessibility, SEO-friendly.

3. Multipage Website

  • Breaks content into multiple pages.
  • Use <a> for navigation.
  • Keep common header & footer for consistency.

4. Grouping & Labeling Elements

  • div: Generic block container.
  • id: Unique identifier for one element.
  • class: Reusable label for multiple elements.
  • span: Inline container for highlighting text.
  • Semantic tags: header, footer, main.
