Lecture 5: HTML Forms

Defence Course - Day 5

Why Forms?

  • Web needs to collect info from users.
  • Without forms → no login, signup, search, comments, etc.
  • <form> solves: display fields, collect + send data.

1. Basic Input

<input type="text">
  • Single-line text box.
  • Use <label> for meaning.
  • Connect label + input with id and for.

2. Submit Data

  • Wrap inputs in <form>.
  • Add <input type="submit"> or <button type="submit">.
  • name attribute = key for submitted data.

3. Radio Buttons (Select One)

<input type="radio" name="shirtSize" value="small">
  • Only one option can be chosen.
  • Same name = one group.
  • Each needs unique id and value.

4. Checkboxes (Select Many)

<input type="checkbox" name="toppings" value="pepperoni">
  • Multiple selections allowed.
  • Same name groups them.
  • Each has unique id and value.

5. Textarea

<textarea rows="4" cols="50"></textarea>
  • Multi-line text input.
  • Use for comments, addresses, etc.

6. Dropdown


<select name="country">
  <option value="in">India</option>
</select>
  • Compact list of many options.
  • value is submitted, text is displayed.

7. HTML5 Inputs

  • type="password" → hides text.
  • type="number" min="18" max="99" → range control.
  • type="date" → calendar picker.
  • Other: color, range, file, placeholder, required.
